# Load Test Simulation

This directory contains scripts and configurations to simulate disk load on the server. This is useful for testing Grafana alerts related to disk usage.

## Files

### 1. `disk_cycle.py`

This Python script simulates disk activity by periodically creating and deleting large files.

*   **Functionality:**
    *   Creates a 2GB file filled with zeros every 30 minutes in the `./disk_test` directory.
    *   Deletes the oldest file every 1 hour and 30 minutes.
    *   This creates a fluctuating disk usage pattern.
*   **Configuration:**
    *   `TARGET_DIR`: Directory where files are created (default: `./disk_test`).
    *   `CREATE_INTERVAL`: Time between file creations (default: 1800 seconds / 30 mins).
    *   `DELETE_INTERVAL`: Time between file deletions (default: 5400 seconds / 1h 30m).

### 2. `grafana-load.service`

This is a Systemd service file to manage the `disk_cycle.py` script as a background service on AlmaLinux (or other systemd-based distributions).

*   **Key Features:**
    *   Runs the script as the `alma` user.
    *   Uses the Python interpreter from a virtual environment (`/home/alma/grafana-ascent/venv/bin/python3`).
    *   Automatically restarts the script if it crashes.
    *   Logs output to the system journal with the identifier `grafana-load`.

## Management Commands (AlmaLinux)

Here are the commands to set up and manage the service on AlmaLinux.

### Prerequisites

Ensure you have Python 3 and the virtual environment set up:

```bash
# Install Python 3 if not present
sudo dnf install python3

# Create virtual environment
cd /home/alma/grafana-ascent
python3 -m venv venv

# Install dependencies (if any are added later, currently none required for standard lib)
# source venv/bin/activate
# pip install -r requirements.txt
```

### Service Setup

1.  **Copy the service file to the systemd directory:**

    ```bash
    sudo cp grafana-load.service /etc/systemd/system/
    ```

2.  **Reload the systemd daemon to recognize the new service:**

    ```bash
    sudo systemctl daemon-reload
    ```

3.  **Enable the service to start automatically on boot:**

    ```bash
    sudo systemctl enable grafana-load
    ```

4.  **Start the service immediately:**

    ```bash
    sudo systemctl start grafana-load
    ```

### Service Management

*   **Check the status of the service:**

    ```bash
    sudo systemctl status grafana-load
    ```

*   **Stop the service:**

    ```bash
    sudo systemctl stop grafana-load
    ```

*   **Restart the service:**

    ```bash
    sudo systemctl restart grafana-load
    ```

### Viewing Logs

You can view the logs generated by the script using `journalctl`:

*   **View all logs for the service:**

    ```bash
    sudo journalctl -u grafana-load
    ```

*   **Follow logs in real-time (like `tail -f`):**

    ```bash
    sudo journalctl -u grafana-load -f
    ```
